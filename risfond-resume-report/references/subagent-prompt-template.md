# 子代理提示词模板（委托 doubao-seed-2-1-pro 撰写）

当使用 `delegate_task` 委托 doubao 撰写报告时，在 goal 和 context 中包含以下要点：

## 必填信息

### 1. 候选人完整资料
- 姓名、性别、年龄、学历、学校、专业
- 籍贯、婚姻、现居、到岗周期
- 每段工作经历：时间、公司、职位、公司介绍、汇报对象、下属人数、工作职责、工作业绩、离职原因
- 项目经历（销售/项目型岗位默认需要）：时间、项目名称、项目角色、项目介绍、项目职责、项目业绩
- 目前薪资、期望薪资

### 2. 核心优势（已确认的定稿版本）
直接粘贴用户已确认的核心优势文本。

### 3. 排版格式要求（逐条列出）
- 中文宋体、西文/数字 Times New Roman，字号10.5pt（五号），行距1.5倍，段前段后0pt
- 页边距上下2.54cm、左右3.17cm
- **基本信息用 4行×2列 无边框表格**（姓名|性别、年龄|学历、籍贯|婚姻、现居|空），不用逐行段落；到岗周期不单独列出，写入【职业状态】
- **所有【】标签都不加粗**，包括【职业经历】【核心优势】【职业状态】【家庭情况】【薪资结构】【意向度】【教育背景】【工作经历】
- 颜色规范：
  - 工作经历/项目经历标题行：Bold + 蓝色#2E75B5（全文仅标题行用彩色）
  - 汇报对象：/ 下属人数：标签：Bold + 黑色#000000
  - 公司介绍：/ 工作职责：/ 工作业绩：/ 项目介绍：/ 项目职责：/ 项目业绩：标签：Bold + 默认黑色，内容不加粗
  - 离职原因/看机会原因标签：Bold + 默认黑色，内容不加粗
  - 其余正文：Normal + 默认黑色
  - 每个文字run都要显式设置 font.color.rgb
- 工作经历/项目经历标题行三段之间用 \t 制表符
- 每段工作经历、每段项目经历之间用空行

### 4. 内容规范
- 推荐顾问固定写 Alfred
- 核心优势每条是一个完整的自然段落，不加粗
- 职业经历时长**紧跟时间后面**，格式如 `2008.12-2026.06（17年6个月） 诺基亚 职位`
  - ❌ 错误：`2008.12-2026.06 诺基亚 职位（17年6个月）`
- 工作职责/工作业绩编号用 1、2、3...
- **已离职人员写"离职原因："**，在职人员写"看机会原因："
- 离职原因标签加粗，内容不加粗
- 尾部声明照抄固定格式
- 薪资格式：XXk*X薪=XXw税前

### 5. 教育背景规范
- 标题图片或【教育背景】文字标题（不加粗）
- 五号字号（10.5pt），行间距1.5倍
- **无段前距，无开头空格**
- 每行一段，格式：YYYY/MM-YYYY/MM  学校  专业  学历(统招/非统招)

### 6. 项目经历规范（销售/项目型岗位默认需要）
- 位于工作经历之后、尾部声明之前，标题图片或【项目经历】文字标题（不加粗）
- 每段项目：标题行（Bold + #2E75B5）`YYYY/MM-YYYY/MM\t项目名称\t项目角色`
- 三个子段：项目介绍：/ 项目职责：/ 项目业绩：，标签加粗、内容不加粗
- 项目业绩必须量化（中标金额/签约额/标杆意义）
- 项目按时间倒序，时间段可与工作经历重叠

### 7. 输出要求
- 用 python-docx 生成脚本
- 西文/数字用 `font.name = 'Times New Roman'`，中文必须用 `qn('w:eastAsia')` 设置为宋体，否则Word渲染为MS明朝
- 保存到 D:\hermes猎头\锐仕方达-候选人姓名-职位.docx
- 只打印"完成"，不要输出其他解释文字

### 8. 标题图片
- 默认使用 skill `assets/` 目录的 4 张标准标题图片（title_basic_info.png / title_education.png / title_work_experience.png / title_project_experience.png）；用户提供参考文件时优先用参考文件中的标题图片
- 不要添加企业logo图片或装饰性分隔线/页眉横幅图片
